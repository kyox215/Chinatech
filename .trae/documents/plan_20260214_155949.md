# UI 优化方案：手机端悬浮报价组件

我将针对手机浏览器优化底部的报价显示，将其从全屏卡片改为紧凑的底部悬浮小组件，以节省屏幕空间。

## 修改 `components/recycling-app.tsx`

1.  **调整布局结构**:
    - 现有的右侧报价栏 (`lg:col-span-4`) 将仅在桌面端显示 (`hidden lg:block`)。
    - 新增一个“手机端悬浮组件”，仅在移动端显示 (`lg:hidden`)。

2.  **创建悬浮组件**:
    - **位置**: 固定在屏幕底部 (`fixed bottom-4 left-4 right-4`)，悬浮于内容之上。
    - **样式**: 紧凑型卡片，带有阴影和背景模糊效果，占用空间极小。
    - **内容**:
        - **左侧**: 突出显示“最终回收价” (例如 `¥ 795`) 和微小的“下月预测”。
        - **右侧**: “确认成交”按钮。
    - **详情展开**:
        - 添加一个小的展开触点 (Chevron Icon)，点击可向上展开查看详细的扣款明细 (基准价、电池损耗等)。

3.  **调整内容区域**:
    - 为主滚动区域增加底部内边距 (`pb-32`)，防止最底部的内容被悬浮组件遮挡。

## 预期效果
- **节省空间**: 悬浮条仅占用约 80px 高度，大幅增加可视内容区域。
- **操作便捷**: 核心信息（价格）和操作（成交）始终在拇指可及范围内。
- **响应式**: 桌面端保持原有的大屏布局，移动端自动切换为悬浮模式。

## 验证
- **手机端**: 确认原来的大卡片消失，底部出现悬浮条。
- **桌面端**: 确认布局未受影响。
- **交互**: 点击悬浮条可展开/收起查看价格明细。
