# Milan Repair Manager 项目初始化计划

我将按照您的 "插拔式架构" (Plug-and-Play) 和 "统一设计语言" (Unified Design Language) 蓝图，为您初始化 Milan Repair Manager 项目。

## 1. 项目初始化与技术栈
- **初始化 Next.js 15**: 使用 App Router, TypeScript, Tailwind CSS。
- **安装依赖**:
  - **UI/图标**: `lucide-react`, `sonner`, `clsx`, `tailwind-merge`, `class-variance-authority`, `radix-ui` 相关组件。
  - **后端/数据**: `prisma`, `@prisma/client`, `next-auth@beta`。

## 2. 设计系统配置 (Milan Industrial)
- **Tailwind 配置**:
  - 颜色: Brand (`Indigo-600`), Surface (`Slate-50/White`), Text (`Slate-900/500`).
  - 字体: `Inter` (界面), `JetBrains Mono` (数据).
  - 圆角: 配置统一的圆角规范。
- **工具类**: 创建 `src/lib/utils.ts` (包含 `cn` 辅助函数)。

## 3. 目录结构实施
我将创建支持 "插拔式" 架构的特定目录结构：
```text
src/
├── modules/
│   ├── shared/ (强制 UI: PageShell, SectionCard)
│   ├── core/   (系统组件: Sidebar, UserMenu)
│   └── repairs/ (示例业务模块)
├── services/   (数据逻辑)
└── components/ui/ (Shadcn 基础组件)
```

## 4. 核心组件开发
- **强制共享组件**:
  - `PageShell.tsx`: 标准化页面外壳 (统一标题、描述、操作栏)。
  - `SectionCard.tsx`: 标准化白色卡片容器。
- **布局组件**:
  - `Sidebar.tsx`: 导航菜单 (Slate-950 风格)。
  - `Header.tsx`: 顶部导航栏。

## 5. 页面与路由设置
- **路由**:
  - `(auth)/login`: 登录页面 UI。
  - `(dashboard)/layout.tsx`: 应用 Sidebar + Header 全局布局。
  - `(dashboard)/page.tsx`: 仪表盘首页 (使用 `PageShell`)。

## 6. 数据库 Schema (Prisma)
- **初始化 Prisma**: 设置 `prisma/schema.prisma`。
- **定义模型**: 实现 `User`, `Ticket`, `Part`, `TicketPart` 及其关联。
- **客户端**: 创建 `src/lib/db.ts` 全局 Prisma 客户端。

## 7. 文档
- 创建 `PROJECT_BLUEPRINT.md` 包含项目愿景和规范。
