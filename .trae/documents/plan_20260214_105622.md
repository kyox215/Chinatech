# 开发回收模块 (ChinaTechOS v1)

我将基于 `/Users/hexiang/Documents/trae_projects/ChinaTechOS v1/回收模块` 的 HTML 逻辑，严格遵循 `UI_DESIGN_GUIDE.md` 规范，在 Next.js 中开发“回收模块”应用。

## 实施计划

1.  **创建页面结构 (Page Structure)**
    *   新建页面路径: `app/dashboard/recycling/page.tsx`。
    *   布局: 采用响应式 Grid 布局，左侧为配置区 (2/3)，右侧为报价与汇总区 (1/3, Sticky)。

2.  **迁移核心逻辑与数据 (Logic & Data Migration)**
    *   移植 `initialCsvData` (iPhone 11-17) 数据和解析逻辑。
    *   实现核心定价算法:
        *   **基准价**: 基础回收价 + 存储容量增值。
        *   **损耗扣款**: 外观成色 (百分比扣款)、电池损耗 (百分比或固定值)、屏幕损坏 (固定值)。
        *   **风控算法 (核心)**: 基于机型发布年份的时间贬值逻辑 (新款掉价快/老款掉价慢) * 持有天数。

3.  **UI 实现 (遵循 UI 设计指南)**
    *   **组件**: 全面使用 `shadcn/ui` 组件 (`Card`, `Select`, `Slider`, `Badge`, `Button`, `ToggleGroup`)。
    *   **色彩**: 使用语义化颜色 (`--primary`, `--destructive` 用于战损/损坏, `--muted` 用于辅助信息)。
    *   **排版**: 使用 `Geist Sans`，遵循 `text-sm`, `font-bold` 等层级规范。
    *   **交互设计**:
        *   **机型选择**: 使用 `Select` 组件。
        *   **存储容量**: 使用 `ToggleGroup` 或分段按钮。
        *   **持有周期**: 使用 `Slider` 滑块，实时反馈贬值金额。
        *   **成色/电池**: 使用可视化卡片选择 (选中时显示 `ring-2 ring-primary`)。
        *   **屏幕状况**: 使用 `Switch` 或 Toggle Card 切换“屏幕损坏”状态。

4.  **本地化 (Localization)**
    *   默认语言设置为 **简体中文** (基于项目上下文及用户偏好)。
    *   将源文件中的术语翻译为专业术语 (如 "Depreciation" -> "库存跌价风险", "Hold Period" -> "预计周转周期")。
