# 创建 UI 设计规范指南 (UI Design Guide)

我将基于当前项目代码库（特别是 `tailwind.config.ts` 和 `app/globals.css`），为您生成一份详尽的 UI 设计规范文档 `UI_DESIGN_GUIDE.md`。该文档将作为未来所有功能开发的 UI 标准。

## 文档内容规划

1.  **核心设计原则 (Core Principles)**
    *   **极简主义 (Minimalist)**: 强调内容，减少装饰。
    *   **一致性 (Consistency)**: 统一的组件库和交互模式。
    *   **现代感 (Modern)**: 使用圆角、阴影和微妙的动画。

2.  **色彩系统 (Color System)**
    *   **语义化颜色**: Primary (主色), Secondary (次色), Destructive (破坏性操作), Muted (柔和), Accent (强调), Popover, Card。
    *   **图表颜色**: 提取 `--chart-1` 到 `--chart-5` 的具体 HSL 值。
    *   **侧边栏专属色**: 详细列出 Sidebar 背景、前景色、选中态等。
    *   **模式支持**: 包含 Light (亮色) 和 Dark (暗色) 两种模式的具体色值定义。

3.  **排版系统 (Typography)**
    *   **字体家族**: Geist Sans (主要无衬线字体) 和 Geist Mono (等宽字体)。
    *   **字号与行高**: 遵循 Tailwind 默认比例。

4.  **布局与间距 (Layout & Spacing)**
    *   **圆角系统 (Radius)**: 统一基准为 `0.5rem` (8px)，衍生出 `lg`, `md`, `sm` 等规格。
    *   **容器宽度**: 定义标准容器的最大宽度。

5.  **组件规范 (Component Standards)**
    *   **卡片 (Card)**: 头部、内容、底部的标准结构。
    *   **按钮 (Button)**: 不同变体 (Default, Outline, Ghost, Link) 的使用场景。
    *   **表单 (Form)**: 输入框、标签的样式规范。

6.  **动画与交互 (Animation)**
    *   **过渡效果**: `accordion-down`, `accordion-up` 等标准动画定义。
    *   **交互反馈**: Hover, Focus 状态的统一处理。

7.  **强制性声明 (Mandatory Declaration)**
    *   文档开头将明确声明：“本项目后续所有功能模块的 UI 设计与开发，必须严格遵循本指南定义的规范与逻辑。”

## 执行步骤
1.  创建 `UI_DESIGN_GUIDE.md` 文件。
2.  将提取的设计 Token 和规范写入文件。
3.  (可选) 如果您需要，我还可以创建一个简单的示例页面来展示这些规范。目前主要聚焦于文档编写。
